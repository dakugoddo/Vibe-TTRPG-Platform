# 🍰 Vertical Slice Planner & PRD Generator

> Когда применять: при ЛЮБОЙ новой фиче или задаче.

---

## 1. ФОРМАТ MINI-PRD

При поступлении запроса на фичу — сгенерируй этот шаблон:

```markdown
# Mini-PRD: [Название фичи]

## Проблема
[1-2 предложения — что сейчас не так]

## Решение
[2-3 предложения — как исправляем]

## User Story
- Как [роль: ГМ/игрок], я хочу [действие], чтобы [ценность]

## Acceptance Criteria
- [ ] Критерий 1
- [ ] Критерий 2
- [ ] Критерий 3

## Edge Cases (TTRPG + технические)
- Случай 1: [описание] → [ожидаемое поведение]
- Случай 2: [описание] → [ожидаемое поведение]

## Технические заметки
- Затрагивает файлы: [список]
- Новые зависимости: [нет / список]
- Риски: [мультиплеер, canvas-синхронизация, перформанс]

## Вертикальные слайсы
### Слайс 1: [MVP — самый тонкий слой]
- DB: [изменения в .md / Yjs]
- Server: [новые endpoint]
- Store: [Zustand]
- UI: [компоненты]
- Tests: [что тестируем]

### Слайс 2+: [дополнения]
...
```

---

## 2. ВЕРТИКАЛЬНЫЙ СЛАЙС

Каждый слайс проходит ВСЕ слои за один проход:

```
DB (.md/Yjs) → Server (REST/WS) → Store (Zustand) → UI (Components) → Tests → Docs
```

### Правила:
1. Слайс 1 = минимально работающая версия (можно потрогать)
2. Не более 3 слайсов на фичу
3. Каждый слайс ≤ 200 строк нового кода
4. Если слайс больше — разбей на под-слайсы

## 3. ПРИМЕР

### Запрос:
> "Хочу, чтобы ГМ мог быстро кидать инициативу для группы монстров"

### Mini-PRD:
```markdown
# Mini-PRD: Group Initiative Roller

## Проблема
ГМ вручную кидает /r 1d20+2 для каждого гоблина. 6 гоблинов = 6 бросков. Скучно.

## Решение
Добавить кнопку "Group Roll" в контекстное меню выделения. Выбрал токены → ПКМ → Бросить инициативу. Все броски в чат одной пачкой.

## User Story
- Как ГМ, я хочу бросать инициативу для группы выделенных токенов, чтобы ускорить начало боя.

## Acceptance Criteria
- [ ] Выделил 3 токена → ПКМ → "Бросить инициативу" → 3 броска в чат
- [ ] Каждый бросок показывает имя существа, значение, модификатор
- [ ] Использует DEX мод существа (useCalculatedStat)
- [ ] Работает с 1 токеном (fallback к обычному броску)
- [ ] Игроки видят результаты (Yjs chat sync)

## Edge Cases
- Существо без DEX стата → мод 0, предупреждение в чат
- 20+ токенов выделено → показываем confirm "точно 20 бросков?"
- Игрок пытается кинуть за чужих существ → только ГМ может

## Технические заметки
- Файлы: InfiniteCanvas.tsx, ChatPanel.tsx, diceParser.ts
- Зависимости: нет
- Риски: спам в чат при большом выделении

## Слайсы
### Слайс 1: Базовый group roll
- Store: groupRoll action в canvasDrawStore
- Math: batchRoll в diceParser
- UI: пункт в контекстном меню выделения
- Chat: batch-сообщение с результатами

### Слайс 2: Полировка
- ConfirmDialog для >20 токенов
- Сортировка по значению инициативы
- Подсветка лидера
```

## 4. КАК ИСПОЛЬЗОВАТЬ

1. Получил запрос от пользователя → спроси уточнения (если нужно)
2. Сгенерируй Mini-PRD
3. Покажи план вертикальными слайсами
4. Спроси: "Начинаю слайс 1?"
5. После каждого слайса → self-critique (см. self-critique skill)
