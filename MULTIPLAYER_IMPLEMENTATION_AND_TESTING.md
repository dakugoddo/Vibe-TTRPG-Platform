# 🌐 План реализации мультиплеера и Undo/Redo (y-websocket) + Тест Гайд

Документ описывает технический план объединения локального (индивидуального) стека Undo/Redo от Yjs с мультиплеерными комнатами `y-websocket`, а также пошаговую инструкцию по тестированию игры с другим человеком по виртуальной локальной сети на базе Hamachi / RadminVPN.

## Часть 1. Архитектурный План Реализации (Технический)

### 1.1 Инструмент Yjs Y.UndoManager
Текущий самописный массив `undoStack` (в Zustand) необходимо заменить или связать с нативным классом `Y.UndoManager` из библиотеки Yjs. 
- **Почему это важно:** `Y.UndoManager` "из коробки" слушает локальные операции (те, что совершил текущий клиент) на конкретном `Y.Map` или `Y.Array`. Если другой игрок меняет цвет квадрата (remote update), `Y.UndoManager` его игнорирует. Когда локальный игрок жмет `Ctrl+Z`, `Y.UndoManager` откатывает только локальную дельту (Selective Undo).

### 1.2 "Режим Редактирования" (Транзишены Канвасов)
Из-за разделения ОЗУ мы загружаем только один канвас за раз (Sub-documents).
**Логика:**
1. В Zustand-стор добавляется флаг `isCanvasDirty: boolean`, который становится `true` при первом локальном экшене в `Y.UndoManager`.
2. Если игрок кликает на Портал (или пытается переключить область в Базе), срабатывает перехватчик:
   ```javascript
   if (isCanvasDirty) {
     const confirmLeave = window.confirm("Вы внесли изменения на этом холсте. При переходе история ваших персональных отмен (Undo) для него будет полностью очищена. Изменения на самом холсте сохранятся. Перейти?");
     if (!confirmLeave) return;
   }
   ```
3. При подтверждении:
   - Клиент отключается от текущей WebSocket комнаты канваса.
   - Локальный `undoManager.clear()` сбрасывает память.
   - `isCanvasDirty` переводится в `false`.
   - Происходит подключение к новой комнате (новому Y.Doc).

## Часть 2. Инструкция по Тестированию Мультиплеера (Для тебя и твоего друга)

> **Примечание:** Следуйте этим шагам ТОЛЬКО ПОСЛЕ того, как фаза внедрения `y-websocket` и `Y.UndoManager` будет завершена в коде.

### Шаг 1: Подготовка Виртуальной Сети (Hamachi / RadminVPN)
1. Убедитесь, что у тебя и у твоего друга установлена одинаковая программа для LAN-эмуляции.
2. Ты (Хост / ГМ) создаешь новую сеть с паролем.
3. Твой друг подключается к этой сети.
4. В приложении Hamachi / RadminVPN рядом с твоим именем появится **IPv4-адрес** (например, `25.55.120.14`). Запиши его, он нужен другу.

### Шаг 2: Запуск Серверов (Делает только Хост)
1. Открой проект в VS Code на своём ПК.
2. Запусти Express-сервер и Клиент:
   - В терминале клиента: `npm run dev -- --host` (флаг `--host` обязателен, чтобы сервер прокинулся в локальную сеть).
   - В терминале сервера: Запусти сервер (например, `npm run start` по порту 3001).
3. Убедись, что твой брандмауэр Windows не блокирует порты `5173` и `3001` (при первом запуске Windows обычно спрашивает разрешение на доступ к сети - разреши для частных сетей).

### Шаг 3: Вход в игру
**Для Тебя (Хоста/ГМа):**
- Открой в своем браузере `http://localhost:5173`.
- Выбери комнату и зайди на холст.

**Для Друга (Клиента):**
- Другу **не нужно** ничего скачивать или запускать терминалы.
- Ему нужно просто открыть любой браузер (Chrome) и ввести в адресную строку: `http://<ТВОЙ_IP_ИЗ_HAMACHI>:5173` (например, `http://25.55.120.14:5173`).
- Друг заполнит никнейм и войдет в ту же самую сессию.

### Шаг 4: Тест №1 — Базовая Синхронизация и Курсоры
1. Ты и твой друг должны видеть перемещение курсоров друг друга (через Yjs Awareness).
2. Нарисуй квадрат. Друг должен мгновенно увидеть его.
3. Друг рисует круг. Ты должен мгновенно увидеть круг.

### Шаг 5: Тест №2 — Локальный Undo/Redo Конфликт (Самый сложный тест)
1. Ты рисуешь новый Квадрат.
2. Скажи другу перекрасить этот Квадрат в **красный цвет**. Он перекрашивает.
3. Теперь ты **передвигаешь** этот же красный квадрат в правый угол экрана.
4. Сразу после этого ТЫ нажимаешь `Ctrl+Z` (Отмена).
   - **Ожидаемый результат:** Квадрат возвращается обратно в центр (отменилось ТВОЕ движение), но он **остается красным** (настройки цвета ДРУГА остались). Selective Undo работает.

### Шаг 6: Тест №3 — Transition & Dirty Флаг
1. Друг делает любую правку (например, дорисовывает линию).
2. Друг пытается кликнуть на Портал, чтобы уйти на другую карту.
3. У Друга должно всплыть окно: `"Вы внесли изменения... История отмен будет очищена."`
4. Друг жмет "ОК". Экран друга загружает новую карту.
5. Нажми `Ctrl+Z` у себя — изменение друга никуда не пропадёт. Пусть Друг вернётся на холст к тебе и нажмет `Ctrl+Z` — ничего не произойдёт, так как его стек Undo был стерт при переходе.
